🎯 Key Innovations

Meta-Programming

PRISM operates at a higher level of abstraction, using meta-programs that represent common problem-solving strategies:

  • Recursive decomposition
  • Symmetry completion
  • Object-wise transformation
  • Pattern propagation
  • Constraint solving

Program Sketches

High-level program structures with slots to be filled:

  • Separates structure from details
  • Enables targeted search
  • Supports compositional reasoning
  • Allows constraint propagation

Adaptive Learning

The MetaLearner component learns from synthesis successes:

  • Feature-based sketch selection
  • Success history tracking
  • Dynamic weight adjustment
  • Exploration vs exploitation

🔧 Meta-Program Types

RECURSIVE_DECOMPOSE

Break complex problems into simpler subproblems, solve recursively, then combine results. Ideal for tasks with regular structure or self-similar patterns.

SYMMETRY_COMPLETE

Identify partial symmetries and complete them. Works well for tasks involving mirror operations, rotational symmetry, or pattern completion.

OBJECT_TRANSFORM

Detect individual objects and apply transformations to each independently. Perfect for tasks where objects maintain their identity but change properties.

PATTERN_PROPAGATE

Extract local patterns and propagate them globally according to rules. Excellent for tiling, repetition, and pattern extension tasks.

CONSTRAINT_SOLVE

Formulate the task as a constraint satisfaction problem and solve iteratively. Useful for tasks with logical rules or interdependencies.

HIERARCHICAL_COMPOSE

Build solutions bottom-up through multiple levels of abstraction. Suitable for tasks requiring staged transformations.
