Training Phases Overview

OLYMPUS training consists of several distinct phases, each with different expectations and success criteria:

Exact Match Injection Stage 0: Foundation Stage 1: Intermediate Stage 2: Advanced

Phase 1: Exact Match Injection

Purpose: Pre-train model to achieve exact matches on simple, deterministic patterns

Target: 95% exact match accuracy

Duration: 30-100 epochs

Example Log Progression

EXACT MATCH INJECTION TRAINING ================================================== Epoch 1/100: Exact Match: 16.0% | Loss: 1.2370 | LR: 0.00005 Epoch 10/100: Exact Match: 35.1% | Loss: 0.4551 | LR: 0.00007 Epoch 20/100: Exact Match: 35.8% | Loss: 0.4450 | LR: 0.00008 Epoch 30/100: Exact Match: 35.6% | Loss: 0.1209 | LR: 0.00010 Epoch 31/100: Exact Match: 33.6% | Loss: 0.5821 | LR: 0.00010

Healthy Progression Patterns

  • Epochs 1-20: Steady climb from ~15% to ~35%
  • Epochs 21-30: Loss drops rapidly as exact match bonus activates
  • Epoch 31+: Expected temporary drop (33-35%) as full loss function engages
  • Learning Rate: Gradual increase during 30-epoch warmup period

Warning Signs to Monitor

  • Plateau below 30% for more than 20 epochs
  • Sudden drops greater than 5% accuracy
  • Loss values becoming NaN or infinite
  • Gradient norms consistently above 50

Stop Training and Adjust If

  • Exact match accuracy stuck below 25% for more than 20 epochs
  • Loss becomes NaN or infinite values appear
  • No improvement whatsoever after 60 epochs
  • GPU memory errors or system instability

Stage 0: Foundation Learning

Purpose: Learn basic ARC patterns and fundamental transformations

Target: 5-15% exact match on real ARC validation data

Duration: 100 epochs per stage

Stage 1: Intermediate Complexity

Purpose: Handle more complex patterns and larger grid sizes

Target: 3-8% improvement in exact match performance

Duration: 100 epochs with reduced resources

Healthy Stage 1 Progress Signs

  • Automatic resource reduction: batch size 512→256, workers 8→4
  • Dataset automatically limited to 10K samples maximum
  • MEPT replay buffer actively engaged and growing
  • Progress bar showing actual movement: Stage 1, Epoch 1/100: 15% 5/36

Decision Tree: When to Intervene

1. Check Current Training Phase

Exact Match Injection Phase:

  • Target: >90% accuracy by epoch 60
  • If stuck <30%: Reduce initial LR to 0.00005
  • If loss explodes: Restart with lower learning rate

2. Identify Common Issues

Progress Bar Completely Stuck:

  • Stage 1+ hanging - automatic fixes should activate
  • If still stuck >10 minutes: Stop and restart training

3. Performance Threshold Checks

Stop Training Immediately If:

  • Exact match <1% for >20 epochs in Stage 0
  • Loss consistently increasing for >15 epochs
  • GPU memory errors or persistent NaN losses

4. System Health Monitoring

MEPT Memory System Status:

  • Buffer should grow each epoch
  • Exact matches >100 by epoch 20
  • Pattern bank >1000 unique patterns

Training Success Criteria

Consider Training Successful When

  • Exact Match Injection: Achieved >90% exact match accuracy
  • Stage 0: 8-15% exact match on validation data
  • Stage 1: 10-20% exact match on validation data
  • Stage 2: 15-25% exact match on validation data
  • MEPT Buffer: >10,000 experiences with >500 exact matches
  • Pattern Bank: >5,000 unique patterns successfully learned
  • Model File: 20-40MB final checkpoint size

Training Complete But Suboptimal When

  • Final exact match between 5-10% (usable but not optimal)
  • Strong pixel accuracy (>80%) but consistently low exact match
  • Good Stage 0 performance but poor Stage 1+ progression
  • MEPT system not properly accumulating diverse patterns

Emergency Troubleshooting

Stop Training Immediately If

  • GPU Memory Error: "CUDA out of memory" messages
  • NaN Cascade: Multiple consecutive NaN loss values
  • Infinite Loss: Loss values exceeding 100 or becoming infinite
  • System Unresponsive: No progress for more than 30 minutes
  • Storage Issues: Cannot save checkpoints due to disk space

Quick Recovery Steps

  1. Clear GPU memory by restarting runtime
  2. Reduce batch size: 512 → 256 → 128
  3. Lower learning rate: 0.005 → 0.002 → 0.001
  4. Resume from last good checkpoint if available
  5. Verify data pipeline integrity

Diagnostic Commands

  • nvidia-smi - Check GPU status
  • df -h - Check disk space
  • ps aux | grep python - Check processes
  • ls -la arc_models_v4/ - Check model files
